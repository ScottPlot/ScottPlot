# Architecture Rules (Non-Obvious)

- **The Render Pipeline is extensible but ordering is CRITICAL** — `RenderManager.RenderActions` is a public `List<IRenderAction>` that can be modified externally. However, the order of the 25 steps is carefully designed. For example, `StoreGLState` MUST run before `RenderGridsBelowPlottables` and `RestoreGLState` MUST run after. Inserting new actions at the wrong position will cause rendering corruption. See [`RenderManager.cs:9`](src/ScottPlot5/ScottPlot5/Rendering/RenderManager.cs:9).
- **Plottables reference Axes by interface, not instance** — each `IPlottable.Axes` holds `IXAxis`/`IYAxis` references. When axes are replaced (e.g., `DateTimeTicksBottom()`), plottable axes are re-assigned. Never cache axis instances across axis-replacement operations.
- **`AxisManager.LinkedAxisRules` sync across Plot instances** — these rules trigger cross-plot axis synchronization via `AxisLimitsChanged` events. To prevent infinite recursion, `DisableAxisLimitsChangedEventOnNextRender` is set on the TARGET plot. If adding new linking logic, you must replicate this guard.
- **Core library MUST NOT reference any UI framework** — `ScottPlot.csproj` has zero UI dependencies. All platform integration lives in `ScottPlot5 Controls/` packages. The core only depends on SkiaSharp and HarfBuzz for cross-platform text shaping.
- **`IPlotControl` is the bridge between Plot and UI** — every UI control implements this interface. The `Plot.PlotControl` property stores a back-reference for calling `Refresh()` on axis changes. This is null for console/headless usage.
- **OpenGL is opt-in and render-pipeline aware** — GL plottables are rendered via the standard pipeline but require state save/restore steps. `IPlottableGL` markers are used to identify GL-bound plottables. The `SyncGLPlottables` step (step 23) synchronizes GL data before the next frame.
- **`MultiplotLayoutSnapshot` enables subplot architectures** — multiple `Plot` instances can be composed into a multiplot view. Axis linking via `LinkedAxisRules` is how subplots share axis limits.
- **Font resolution has a fallback chain** — the font system tries `SystemFontResolver` first, then falls back to `FileFontResolver`. This is critical for Linux/headless deployments where system fonts may be unavailable. See [`ScottPlot5/Fonts/`](src/ScottPlot5/ScottPlot5/Fonts/).
